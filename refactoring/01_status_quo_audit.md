# 1. 现状痛点清单与审计 (Status Quo Audit)

基于对现有 `lemon` 系统代码库的深度审计，结合现代企业级架构标准，我们识别出以下 15+ 核心痛点，涵盖性能、安全、可维护性和扩展性四个维度。

## 1.1 现状痛点清单

### **性能维度 (Performance)**
1.  **阻塞式 I/O 模型**: 现有系统基于传统的 Servlet 3.0 + Spring MVC 4.x，缺乏非阻塞 I/O 支持。在高并发场景下（如大规模审批流转），线程池极易耗尽，导致系统假死。
2.  **数据库连接池瓶颈**: 使用过时的 `commons-dbcp` 1.4，缺乏现代连接池（如 HikariCP）的性能优化，连接泄露风险高，且在高负载下争用严重。
3.  **缺乏有效缓存策略**: Hibernate 二级缓存被禁用或配置不当，且未集成 Redis 集群。大量读多写少的业务数据（如组织架构、数据字典）直接穿透到数据库，造成 IO 压力。
4.  **N+1 查询泛滥**: `HibernateEntityDao` 和 `HibernatePagingDao` 的通用查询缺乏 `JOIN FETCH` 策略，导致在列表页展示复杂关联数据时触发大量 SQL 查询。
5.  **静态资源加载缓慢**: 前端资源未通过 CDN 分发，且缺乏 HTTP/2 和 Gzip/Brotli 压缩支持，首屏加载时间（FCP）过长。

### **安全维度 (Security)**
6.  **弱密码哈希算法**: 用户密码存储使用 `MD5 + 固定盐值`，极易被彩虹表破解，不符合 NIST 安全标准（推荐 Argon2id 或 BCrypt）。
7.  **硬编码敏感信息**: 数据库连接、邮件服务器密码等敏感配置硬编码在 `application.properties` 或 XML 文件中，存在严重的泄露风险。
8.  **CSRF/XSS 防护不足**: 缺乏统一的 CSRF Token 校验机制，且 JSP 页面中存在大量未转义的输出（User Input），易受 XSS 攻击。
9.  **依赖库漏洞**: 依赖大量已停止维护的库（如 `struts2` 插件残留、`commons-fileupload` 旧版本），存在已知的 RCE 漏洞风险。

### **可维护性维度 (Maintainability)**
10. **单体巨石架构**: 所有业务模块（BPM, CMS, Auth, Disk）耦合在同一个 WAR 包中，修改一个模块可能导致整个系统崩溃，无法独立部署和扩展。
11. **贫血模型 (Anemic Domain Model)**: 业务逻辑分散在 Service 层和 Controller 层，实体类仅为数据容器（Getter/Setter），缺乏领域逻辑封装，违反 DDD 原则。
12. **测试覆盖率低**: 单元测试严重缺失，现有测试多为依赖数据库的集成测试，运行缓慢且不稳定（Flaky Tests），导致重构风险极高。
13. **JSP 逻辑耦合**: 视图层包含大量 Java 代码（Scriptlets）和复杂的 Taglib 逻辑，前后端深度耦合，难以迁移至现代前端框架（Vue/React）。

### **扩展性维度 (Scalability)**
14. **同步调用链**: 模块间调用（如审批触发消息通知）均为同步阻塞调用，缺乏事件驱动机制（Event Bus），导致系统吞吐量受限。
15. **水平扩展困难**: 依赖 HttpSession 存储用户状态（Stateful），导致无法平滑地进行水平扩展（Scale-out），必须依赖粘性会话（Sticky Session）。
