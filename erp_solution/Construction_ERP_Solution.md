# 建筑工程企业级 ERP 管理系统完整解决方案

## 1. 精准需求与痛点解决映射

| 当前痛点 | 解决方案规划 | 核心业务价值 |
| :--- | :--- | :--- |
| **流程混乱，合同签证无留痕** | 引入 BPMN 2.0 引擎，实现合同签订、变更、签证、结算全生命周期版本控制与电子签名。 | 合规防险，追溯问责。 |
| **数据不通，进度与成本脱节** | 建立“项目 WBS（工作分解结构）与 CBS（成本分解结构）”映射基准，实现“量、价、费”三维联动。 | 动态掌握利润池，防止超支。 |
| **材料采购对账难，库存不明** | 扫码入库 + 供应商协同门户（B2B）。采购申请自动比对预算，超预算触发熔断或特批。 | 堵住材料流失黑洞，降低采购成本。 |
| **权限不清，数据越权可见** | RBAC（基于角色的访问控制） + ABAC（基于属性的数据权限），按项目、组织架构多维隔离。 | 满足多项目部独立核算、集团统管的诉求。 |
| **报表滞后易错** | BI 报表引擎实时抽取业务数据，生成“管理驾驶舱”（Gantt 图、S 曲线、资金漏斗）。 | 辅助高层实时决策。 |
| **现场数据无法回传，移动端不友好**| 推出适配移动端（微信/钉钉小程序）的“现场助手”，支持拍照上传、定位打卡、隐患工单闭环。 | 提升现场管理效率与真实性。 |

## 2. 系统模块架构图 (Module Architecture)

采用“大中台，小前台”理念，分为核心业务域与支撑域：

*   **门户层 (Portal)**: 集团大屏驾驶舱、项目经理工作台、供应商协同门户、移动端现场助手。
*   **业务应用层 (Business Apps)**:
    *   **项目管理 (PM)**: 立项审批、WBS 计划、进度填报、Gantt 图。
    *   **合同管理 (CM)**: 承包合同、分包合同、变更签证、产值申报。
    *   **成本管理 (Cost)**: 目标成本测算、动态成本监控、资金计划。
    *   **物资设备 (SCM)**: 需用计划、采购寻源、入库领用、盘点调拨、设备台账。
    *   **质量安全 (QHSE)**: 安全巡检、整改通知单、质量验收。
*   **基础支撑层 (Foundation)**:
    *   工作流引擎 (Workflow)、文档中心 (DMS)、组织权限 (IAM)、报表中心 (BI)、消息中心。

## 3. 角色与权限设计 (RBAC + Data Scope)

*   **公司高管 (C-Level)**: 全局数据只读，看经营驾驶舱，审批重大超支及超限额合同。
*   **项目经理 (PM)**: 拥有其负责项目的完全读写权限。核心职责：进度把控、成本动态监控、分包签证审批。
*   **采购经理**: 全局物料库维护，供应商准入，采购合同签订。
*   **预算专员/造价师**: CBS 成本基准编制，产值核算，工程量清单录入。
*   **现场施工员/安全员**: 移动端为主。填报施工日志，发起质量安全整改单。
*   *数据权限规则*: 任何查询必须携带 `projectId` 或 `tenantId` 过滤，严禁跨项目越权。

## 4. 核心业务流程 (Core Business Flows)

### 4.1 动态成本管控闭环
1.  **预算录入**: 中标后，造价师录入《目标成本指导书》（量价分离）。
2.  **业务发生**: 发生采购、劳务分包时，系统自动拦截：若 `累计已发生额 + 本次申请额 > 目标成本`，强制走超支特批流。
3.  **动态呈现**: 实时计算“预计总成本 = 已发生成本 + 待发生成本”，预警利润风险。

### 4.2 签证与变更闭环
1.  **发起**: 现场人员拍照上传《工程变更联系单》。
2.  **审批**: 项目经理 -> 公司工程部 -> 造价部（评估造价影响）。
3.  **归档**: 审批通过后，自动生成补充协议，并**动态调整项目总预算基准 (CBS)**。

## 5. 数据库核心设计 (MySQL 8)

*   **`sys_user`, `sys_role`, `sys_org`**: 基础权限表。
*   **`pm_project`**: 项目主表（状态：投标中、筹备中、施工中、已竣工、已结算）。
*   **`pm_wbs_task`**: 进度计划表（树形结构，记录计划起始时间、实际起始时间）。
*   **`cost_budget_item`**: 成本预算科目表（CBS树，绑定项目ID，记录目标量、目标价）。
*   **`cm_contract`**: 合同主表（类型：主合同、分包合同、采购合同）。
*   **`scm_material_stock`**: 物资库存明细表（记录仓库、物料SKU、数量）。

*关键设计*: 使用 `Snowflake` 算法生成分布式主键；金额统一使用 `DECIMAL(15,2)`；所有核心表增加 `create_time`, `update_time`, `create_by`, `is_deleted` (软删除)。

## 6. 前后端技术架构 (Tech Stack)

*   **前端 (Frontend)**:
    *   Web端: **Vue 3 + TypeScript + Vite + Element Plus / TDesign**。使用 Pinia 管理状态。
    *   可视化: **ECharts** (大屏驾驶舱), **dhtmlxGantt** (项目甘特图)。
    *   移动端: **Uni-App** (一次开发，编译为微信小程序与 App)。
*   **后端 (Backend)**:
    *   框架: **Java 21 + Spring Boot 3.2** (响应式或虚拟线程提升并发)。
    *   微服务/模块化: 采用 **DDD (领域驱动设计)**，初期单体模块化 (Modulith)，后期可平滑拆分 Spring Cloud。
    *   ORM: **MyBatis-Plus**。
    *   工作流: **Camunda 7/8** 或 **Flowable** (替代老旧的 Activiti 5)。
*   **中间件 (Middleware)**:
    *   缓存: **Redis Cluster** (字典表缓存、分布式锁)。
    *   存储: **MinIO / 阿里云 OSS** (图纸、BIM模型、合同附件海量存储)。
*   **部署 (DevOps)**:
    *   **Docker + Kubernetes**，Jenkins / GitLab CI 自动化流水线。

## 7. 关键页面 UI/UX 逻辑

1.  **项目全景工作台 (Project 360 View)**
    *   *左侧*: 树状显示当前项目的 WBS 结构。
    *   *右侧大屏*: 上方 KPI 卡片（合同总额、已收回款、累计发生成本、当前利润率）；中间 Gantt 图；下方风险预警列表。
2.  **物资采购对账台 (Material Reconciliation)**
    *   双表比对设计：左侧为采购订单清单，右侧为实际入库单清单。支持“一键勾选对账”，自动生成应付账款 (AP) 凭证。
3.  **移动端现场巡检 (Mobile QHSE)**
    *   大按钮设计，支持离线缓存（现场可能无网）。拍照自动打上时间+GPS水印（防作弊）。

## 8. API 规范与问题解决方案

*   **API 规范**: RESTful 风格，OpenAPI 3.0 文档。统一返回结构 `{ code: 200, data: {...}, message: "success" }`。
*   **高并发审批锁单问题**: 使用 Redis 分布式锁 `Redisson`，防止同一单据被多人同时点击审批通过。
*   **大数据量报表卡顿**: 将实时性要求不高的重度报表（如月度资金结转）移至夜间定时任务 (XXL-JOB) 计算，结果存入统计宽表。
*   **复杂查询 N+1 问题**: 禁用 Hibernate 的延迟加载滥用，采用 MyBatis-Plus 的复杂 SQL 联表查询，或者引入 Elasticsearch 做项目全局搜索。
